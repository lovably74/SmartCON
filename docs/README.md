# 문서 관리 규칙 (Document Management Rules)

## 문서 저장 위치

- **모든 문서는 `docs/` 폴더에 저장합니다.**
- 문서는 루트 디렉토리가 아닌 `docs/` 폴더 하위에만 생성/저장합니다.

## 문서 업데이트 시 백업 규칙

### 백업 폴더
- 문서 업데이트 시 이전 버전은 `docs/backup/` 폴더에 저장합니다.

### 백업 파일명 규칙
- 백업 파일명 형식: `{원본파일명}_v{버전번호}_{YYYYMMDD}.md`
- 예시: `PRD_v1.0_20251213.md`

### 중복 방지
- 버전 정보와 날짜를 포함하여 중복되지 않도록 합니다.
- 동일한 버전과 날짜의 백업이 이미 존재하는 경우, 시간 정보를 추가합니다.
  - 예시: `PRD_v1.0_20251213_104530.md` (시간: 10:45:30)

## 문서 업데이트 프로세스

1. **기존 문서 확인**: `docs/` 폴더에서 업데이트할 문서 확인
2. **백업 생성**: `docs/backup/` 폴더에 이전 버전 백업
3. **문서 업데이트**: `docs/` 폴더의 원본 문서 수정
4. **변경 이력 업데이트**: 문서 내 "변경 이력" 섹션에 새 버전 정보 추가

## 예시

### 초기 문서 생성
```
docs/
  └── PRD.md (v1.0)
```

### 문서 업데이트 (v1.0 → v1.1)
```
docs/
  ├── PRD.md (v1.1) ← 업데이트된 문서
  └── backup/
      └── PRD_v1.0_20251213.md ← 이전 버전 백업
```

### 추가 업데이트 (v1.1 → v1.2)
```
docs/
  ├── PRD.md (v1.2) ← 최신 버전
  └── backup/
      ├── PRD_v1.0_20251213.md
      └── PRD_v1.1_20251214.md ← 이전 버전 백업
```

## 주의사항

- 백업 파일은 삭제하지 않습니다. (변경 이력 추적을 위해)
- 문서 버전은 시맨틱 버저닝(Semantic Versioning)을 따릅니다.
  - Major.Minor.Patch (예: 1.0.0, 1.1.0, 2.0.0)
- 문서 내부에도 버전 정보를 명시합니다.
