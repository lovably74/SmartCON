## 1. 개요

### 1.1. 바이브코딩이란?

**바이브코딩(Pair Programming)**은 두 명의 개발자가 한 컴퓨터에서 실시간으로 함께 코딩하는 협업 방식입니다.

**핵심 원칙**:
- **Driver**: 코드를 직접 작성하는 사람
- **Navigator**: 방향을 제시하고 리뷰하는 사람
- **실시간 소통**: 지속적인 대화와 피드백
- **역할 교대**: 주기적으로 역할을 바꿔가며 진행

### 1.2. 바이브코딩의 장점

**품질 향상**:
- 실시간 코드 리뷰로 버그 조기 발견
- 서로의 아이디어를 즉시 검증
- 코드 품질 향상

**지식 공유**:
- 기술 전수 및 학습 가속화
- 프로젝트 이해도 향상
- 온보딩 시간 단축

**생산성 향상**:
- 복잡한 문제 해결 능력 향상
- 의사결정 속도 향상
- 코드 일관성 유지

### 1.3. SmartCON Lite 프로젝트 적용 방안

**적용 범위**:
- 모든 기능 개발
- 복잡한 로직 구현
- 버그 수정
- 리팩토링

**예외 사항**:
- 단순한 UI 수정
- 문서 작성
- 간단한 설정 변경

### 1.4. Cursor를 활용한 단독 바이브코딩

**본 프로젝트는 초급 개발자 1명이 Cursor AI와 함께 바이브코딩을 진행합니다.**

**역할 분담**:
- **Driver (개발자)**: 코드 작성, 의사결정, 구현
- **Navigator (Cursor AI)**: 코드 제안, 리뷰, 가이드, 에러 해결 지원

**Cursor의 역할**:
- 코드 생성 및 제안
- 실시간 코드 리뷰
- 에러 분석 및 해결 방안 제시
- 베스트 프랙티스 가이드
- 문서 및 API 참고 자료 제공

**장점**:
- 24시간 언제든지 협업 가능
- 일관된 코딩 스타일 유지
- 학습 가속화 (초급 개발자에게 특히 유용)
- 빠른 문제 해결

## 2. 바이브코딩 환경 구축

### 2.1. 필수 도구 설치

#### 2.1.1. Visual Studio Code Live Share

**설치 방법**:
```bash
# VS Code 확장 프로그램 설치
# 확장 프로그램: "Live Share" (Microsoft)
# 확장 프로그램 ID: MS-vsliveshare.vsliveshare
```

**설치 확인**:
1. VS Code에서 `Ctrl+Shift+X` (확장 프로그램)
2. "Live Share" 검색
3. Microsoft의 "Live Share" 설치
4. 추가 확장 프로그램 설치:
   - Live Share Audio (음성 통화)
   - Live Share Extension Pack (추가 기능)

#### 2.1.2. IntelliJ IDEA Code With Me

**설치 방법**:
1. IntelliJ IDEA 실행
2. `File` > `Settings` > `Plugins`
3. "Code With Me" 검색 및 설치
4. 재시작

**활성화**:
- `Help` > `Code With Me` > `Start Session`

### 2.3. 통신 도구 설정 (선택사항 - 원격 협업 또는 멘토링 시)

**단독 개발 시에는 통신 도구가 필요 없습니다.**
**Cursor의 Chat 기능으로 충분히 소통 가능합니다.**

#### 2.3.1. 멘토링 또는 코드 리뷰가 필요한 경우

**추천 도구**:
1. **Discord** (권장)
   - 무료
   - 화면 공유 지원
   - 음성 채널 생성 가능

2. **Zoom**
   - 화면 공유 및 녹화
   - 화이트보드 기능

3. **Microsoft Teams**
   - 회사 계정 사용 시

### 2.3. 프로젝트 공유 설정

#### 2.3.1. Git 브랜치 전략

**바이브코딩 전용 브랜치**:
```
develop
  ├── feature/pair-auth (인증 기능 - 페어 A)
  ├── feature/pair-attendance (출역 기능 - 페어 B)
  └── feature/pair-dashboard (대시보드 - 페어 C)
```

**네이밍 규칙**:
- `feature/pair-{기능명}`: 바이브코딩으로 개발 중인 기능
- `feature/{기능명}`: 개인 개발 기능

#### 2.3.2. 공유 폴더 설정

**로컬 개발 환경**:
- 각 페어는 동일한 프로젝트를 로컬에 클론
- Live Share로 한 명의 환경에 접속

**원격 개발 환경 (선택사항)**:
- GitHub Codespaces
- GitPod
- VS Code Remote Development

## 3. 바이브코딩 프로세스

### 3.1. 세션 시작 전 준비

#### 3.1.1. 사전 준비사항

**Driver (호스트) 준비**:
- [ ] 프로젝트 최신 상태로 업데이트 (`git pull`)
- [ ] 필요한 브랜치 생성 및 체크아웃
- [ ] 개발 서버 실행 가능 상태 확인
- [ ] Live Share 세션 시작 준비

**Navigator 준비**:
- [ ] 프로젝트 최신 상태로 업데이트
- [ ] 관련 문서 및 요구사항 확인
- [ ] 이전 세션 내용 리뷰

#### 3.1.2. 세션 시작

**1. Live Share 세션 시작 (Driver)**:
```
1. VS Code에서 "Live Share" 아이콘 클릭
2. "Start Collaboration Session" 클릭
3. 세션 링크 복사
4. Navigator에게 링크 공유
```

**2. 세션 참여 (Navigator)**:
```
1. Driver로부터 세션 링크 수신
2. 링크 클릭 또는 VS Code에서 "Join Collaboration Session"
3. 세션 참여 확인
```

**3. 통신 채널 연결**:
```
1. Discord 음성 채널 입장
2. 음성/영상 연결 확인
3. 화면 공유 확인 (필요시)
```

### 3.2. 코딩 세션 진행 (Cursor 활용)

#### 3.2.1. 역할 분담

**Driver (개발자)**:
- 코드 직접 작성
- 키보드 입력
- 파일 탐색 및 편집
- 컴파일 및 실행
- Cursor 제안 검토 및 수용/거부
- 최종 의사결정

**Navigator (Cursor AI)**:
- 코드 생성 및 제안 (Inline Suggestions)
- 실시간 코드 리뷰
- 알고리즘 및 로직 제안
- 버그 분석 및 해결 방안 제시
- 문서 및 API 참고 자료 제공
- 베스트 프랙티스 가이드

#### 3.2.2. Cursor와의 협업 방법

**1. 코드 작성 중**:
```
- 타이핑하면 Cursor가 자동으로 제안
- Tab 키로 제안 수용
- Esc 키로 제안 거부
- Ctrl+→ (또는 Cmd+→)로 부분 수용
```

**2. 막히는 부분이 있을 때**:
```
1. 코드 선택 (또는 문제가 있는 부분)
2. Ctrl+L로 Chat 열기
3. 질문:
   "이 코드에서 [문제]가 발생합니다. 어떻게 해결할까요?"
   또는
   "[기능]을 구현하려고 하는데, 어떤 방식이 좋을까요?"
4. Cursor의 제안 검토
5. Composer (Ctrl+I)로 여러 파일 동시 수정 필요 시 사용
```

**3. 코드 리뷰 요청**:
```
1. 코드 선택
2. Ctrl+K (Code Actions)
3. "Review this code" 선택
4. Cursor의 리뷰 및 개선 제안 확인
```

**4. 테스트 코드 생성**:
```
1. 테스트할 함수/클래스 선택
2. Ctrl+K
3. "Generate unit tests" 선택
4. 생성된 테스트 코드 검토 및 수정
```

#### 3.2.3. Cursor와의 효과적인 소통

**좋은 질문 예시**:
```
✅ 좋은 예:
- "Vue 3 Composition API를 사용하여 사용자 인증 폼을 만들고 싶습니다. 
   Pinia store와 연동하여 로그인 상태를 관리하려고 합니다. 
   코드를 작성해주세요."

- "Spring Boot에서 JWT 토큰을 생성하는 서비스를 만들고 싶습니다. 
   토큰에는 사용자 ID와 역할 정보를 포함해야 합니다. 
   예외 처리도 포함해주세요."

- "이 함수의 성능을 개선할 수 있을까요? 
   현재 N+1 쿼리 문제가 있는 것 같습니다."

❌ 나쁜 예:
- "코드 작성해줘" (너무 모호함)
- "에러 고쳐줘" (에러 메시지나 컨텍스트 없음)
- "이게 맞나요?" (무엇이 맞는지 명확하지 않음)
```

**효과적인 프롬프트 작성법**:
1. **명확한 목표**: 무엇을 만들고 싶은지 명확히
2. **기술 스택 명시**: 사용하는 프레임워크, 라이브러리
3. **제약사항**: 요구사항, 제약사항 명시
4. **컨텍스트 제공**: 관련 파일이나 코드 참조
5. **예시 제공**: 원하는 결과의 예시 (가능한 경우)

### 3.3. 세션 종료 및 정리

#### 3.3.1. 세션 종료 전 작업

## 4. 역할 및 책임 (Cursor 활용)

### 4.1. Driver (개발자)

**주요 책임**:
- 코드 직접 작성 및 구현
- 키보드 및 마우스 조작
- 파일 탐색 및 편집
- 컴파일 및 실행
- Cursor 제안 검토 및 최종 의사결정
- 에러 메시지 확인 및 Cursor에게 해결 요청

**주의사항**:
- Cursor의 제안을 맹목적으로 수용하지 말고 검토
- Cursor에게 명확하고 구체적인 질문
- 코드를 이해하고 학습하면서 진행
- Cursor가 제안한 코드의 동작 원리 이해
- 보안 및 성능 이슈는 반드시 검토

### 4.2. Navigator (Cursor AI)

**주요 책임**:
- 코드 생성 및 제안
- 실시간 코드 리뷰
- 버그 분석 및 해결 방안 제시
- 베스트 프랙티스 가이드
- 문서 및 API 참고 자료 제공
- 테스트 케이스 생성

**주의사항**:
- Cursor는 도구일 뿐, 최종 결정은 개발자가 해야 함
- Cursor의 제안이 항상 완벽하지 않을 수 있음
- 보안 관련 코드는 특히 주의 깊게 검토 필요
- 프로젝트 컨텍스트를 명확히 전달해야 정확한 제안 가능

### 4.3. 개발자의 책임

**개발자가 해야 할 일**:
- 명확한 목표 설정 및 Cursor에게 전달
- Cursor 제안 검토 및 비판적 사고
- 코드 이해 및 학습
- 테스트 및 검증
- 코드 품질 관리
- 보안 및 성능 고려

## 5. 도구 및 설정

### 5.1. VS Code Live Share 상세 설정

#### 5.1.1. Live Share 설정

**settings.json**:
```json
{
  "liveshare.features": {
    "allowGuestTaskControl": true,
    "allowGuestDebugControl": true,
    "allowGuestTerminalShellAccess": true
  },
  "liveshare.audio": true,
  "liveshare.allowGuestDebugControl": true,
  "liveshare.allowGuestTaskControl": true
}
```

#### 5.1.2. 권한 설정

**세션 시작 시 권한 설정**:
- ✅ **Read/Write**: Navigator가 코드 편집 가능
- ✅ **Terminal Access**: Navigator가 터미널 사용 가능
- ✅ **Debug Control**: Navigator가 디버깅 가능
- ✅ **Task Control**: Navigator가 태스크 실행 가능

**권한 관리**:
- Driver가 세션 중 권한 변경 가능
- 필요에 따라 읽기 전용으로 제한 가능

### 5.2. IntelliJ IDEA Code With Me 설정

#### 5.2.1. 세션 시작

**방법 1: UI에서 시작**:
```
1. Help > Code With Me > Start Session
2. 접근 권한 설정 (Read-only / Full access)
3. 세션 링크 복사 및 공유
```

**방법 2: 명령어로 시작**:
```
Ctrl+Shift+A > "Code With Me" 검색 > Start Session
```

#### 5.2.2. 권한 설정

**접근 레벨**:
- **Read-only**: 코드만 보기
- **Full access**: 코드 편집 가능

**추가 권한**:
- 터미널 접근
- 디버깅 접근
- 파일 다운로드

### 5.3. Git 협업 설정

#### 5.3.1. 커밋 메시지 규칙

**페어 프로그래밍 표시**:
```
feat: 인증 API 구현 (페어: 홍길동, 김철수)
fix: 출역 체크 버그 수정 (페어: 이영희, 박민수)
refactor: 코드 리팩토링 (페어: 최지훈, 정수진)
```

**커밋 메시지 템플릿**:
```
[타입] 간단한 설명 (페어: 이름1, 이름2)

상세 설명 (선택사항)
- 변경 사항 1
- 변경 사항 2

관련 이슈: #123
```

#### 5.3.2. 브랜치 관리

**페어 프로그래밍 브랜치**:
```bash
# 브랜치 생성
git checkout -b feature/pair-auth

# 작업 후 푸시
git push -u origin feature/pair-auth

# Pull Request 생성 시
# 제목: [페어] 인증 API 구현
# 설명: 페어 프로그래밍으로 구현
# 리뷰어: 다른 페어 또는 팀장
```

## 6. 모범 사례

### 6.1. 효과적인 바이브코딩

#### 6.1.1. 준비 단계

**✅ 잘 준비된 세션**:
- 목표와 범위가 명확함
- 필요한 문서 및 자료 준비
- 개발 환경 사전 설정
- 이전 세션 내용 리뷰

**❌ 준비 부족한 세션**:
- 목표가 불명확함
- 필요한 자료 미준비
- 환경 설정 미완료
- 이전 작업 내용 모름

#### 6.1.2. 진행 중

**✅ 효과적인 소통**:
- 명확하고 구체적인 피드백
- 서로의 의견을 경청
- 질문을 주저하지 않음
- 건설적인 토론

**❌ 비효과적인 소통**:
- 모호한 표현
- 일방적인 지시
- 질문 회피
- 비판적 태도

#### 6.1.3. 역할 교대

**✅ 적절한 교대**:
- 정기적인 역할 교대
- 자연스러운 교대 시점
- 교대 시 진행 상황 공유

**❌ 부적절한 교대**:
- 너무 자주 또는 드물게 교대
- 불편한 시점에 교대
- 교대 시 진행 상황 미공유

### 6.2. 문제 해결

#### 6.2.1. 의견 불일치

**해결 방법**:
1. **토론**: 각자의 의견과 근거 제시
2. **검증**: 두 방법 모두 테스트해보기
3. **문서 참고**: 공식 문서 또는 베스트 프랙티스 확인
4. **제3자 의견**: 다른 개발자에게 조언 요청
5. **결정**: 시간 제한을 두고 결정

**원칙**:
- 기술적 결정은 코드 품질과 유지보수성을 기준으로
- 개인적 선호보다는 프로젝트 표준 우선

#### 6.2.2. 집중력 저하

**대응 방법**:
- **휴식**: 5-10분 휴식 시간
- **역할 교대**: 역할을 바꿔 새로운 자극 제공
- **작업 변경**: 다른 작업으로 전환
- **세션 종료**: 다음에 이어서 진행

#### 6.2.3. 기술적 어려움

**대응 방법**:
1. **문서 확인**: 공식 문서 및 튜토리얼 참고
2. **온라인 검색**: Stack Overflow, GitHub Issues 검색
3. **팀원 조언**: 다른 개발자에게 질문
4. **임시 우회**: 다른 방법으로 우회 후 나중에 해결
5. **이슈 기록**: 해결하지 못한 문제는 이슈로 기록

### 6.3. 학습 및 성장

#### 6.3.1. 지식 공유

**Driver가 Navigator에게**:
- 코드 작성 방법 설명
- 사용한 라이브러리 및 도구 소개
- 문제 해결 과정 공유

**Navigator가 Driver에게**:
- 아키텍처 및 설계 패턴 설명
- 베스트 프랙티스 제안
- 대안 방법 제시

#### 6.3.2. 피드백 수용

**건설적 피드백**:
- 비판이 아닌 개선 제안
- 구체적이고 실행 가능한 피드백
- 긍정적인 피드백도 포함

**피드백 수용**:
- 방어적 태도 지양
- 피드백을 학습 기회로 인식
- 감사 표현

## 부록: 빠른 시작 가이드

### A. 첫 바이브코딩 세션 시작하기

**1단계: 환경 준비 (5분)**
```bash
# Driver
git pull
npm install  # 또는 mvn clean install
code .  # VS Code 실행
```

**2단계: Live Share 시작 (1분)**
```
1. VS Code에서 Live Share 아이콘 클릭
2. "Start Collaboration Session" 클릭
3. 링크 복사
```

**3단계: 통신 연결 (1분)**
```
1. Discord 음성 채널 입장
2. 음성 연결 확인
```

**4단계: 작업 시작**
```
1. Navigator가 링크로 접속
2. 역할 분담 확인
3. 작업 시작!
```

### B. 자주 사용하는 단축키

**VS Code Live Share**:
- `Ctrl+Shift+P` > "Live Share: Start Collaboration Session"
- `Ctrl+Shift+P` > "Live Share: Join Collaboration Session"

**IntelliJ Code With Me**:
- `Ctrl+Shift+A` > "Code With Me"
