# 스마트콘 라이트(SmartCON Lite) UI 디자인 가이드

**버전:** 1.0  
**작성일:** 2025년 12월 18일  
**작성자:** Manus AI

---

## 1. 개요 (Overview)

본 문서는 건설 현장 인력 관리 SaaS 플랫폼 **'스마트콘 라이트(SmartCON Lite)'**의 사용자 인터페이스(UI) 디자인 및 구현을 위한 가이드라인입니다. 본사 관리자, 현장 관리자, 노무자, 슈퍼 어드민 등 다양한 사용자 그룹에게 일관된 경험을 제공하고, PC부터 모바일까지 모든 디바이스에서 최적화된 사용성을 보장하는 것을 목표로 합니다.

---

## 2. 디자인 시스템 (Design System)

### 2.1. 컬러 팔레트 (Color Palette)

신뢰감과 전문성을 전달하는 **Dark Navy**를 주조색으로, 명확한 인지와 행동 유도를 위한 **Teal**과 **Blue**를 보조색으로 사용합니다.

| 구분 | 색상명 | Hex Code | 용도 |
| :--- | :--- | :--- | :--- |
| **Primary** | **Dark Navy** | `#0F172A` | 헤더, 사이드바, 주요 텍스트, 브랜드 아이덴티티 |
| **Secondary** | **Teal Green** | `#10B981` | 성공 상태, 강조 버튼, 긍정적 지표 (출역 마감 등) |
| **Accent** | **Sky Blue** | `#0EA5E9` | 링크, 활성 상태, 정보성 알림 |
| **Background** | **Light Gray** | `#F8FAFC` | 전체 배경색 (눈의 피로도 감소) |
| **Surface** | **White** | `#FFFFFF` | 카드, 모달, 컨텐츠 영역 배경 |
| **Error** | **Red** | `#EF4444` | 오류, 삭제, 위험 상태 (결근, 계약 만료 등) |
| **Warning** | **Orange** | `#F97316` | 경고, 대기 상태 (지각, 서명 대기) |

### 2.2. 타이포그래피 (Typography)

가독성이 뛰어나고 현대적인 **Pretendard** 폰트를 기본으로 사용합니다.

*   **Font Family**: `Pretendard`, `-apple-system`, `BlinkMacSystemFont`, `system-ui`, `Roboto`, `sans-serif`
*   **Scale**:
    *   **H1 (Page Title)**: 24px (Bold) / Mobile: 20px
    *   **H2 (Section Title)**: 20px (SemiBold) / Mobile: 18px
    *   **H3 (Card Title)**: 16px (SemiBold)
    *   **Body (Default)**: 14px (Regular)
    *   **Caption**: 12px (Regular, Gray-500)

### 2.3. 아이콘 (Iconography)

*   **Library**: `Lucide React`
*   **Style**: Stroke width 2px, Rounded corners
*   **Size**: Default 16px~20px, Button Icon 16px

## 3. 레이아웃 및 그리드 (Layout & Grid)

### 3.1. 반응형 브레이크포인트 (Breakpoints)

Tailwind CSS 표준을 따르며, 모바일 우선(Mobile First) 전략을 적용합니다.

| 디바이스 | Breakpoint | 레이아웃 특징 |
| :--- | :--- | :--- |
| **Mobile** | `< 640px` | 1단 그리드, 하단 탭 바, 햄버거 메뉴, 카드 리스트 뷰 |
| **Tablet** | `640px ~ 1024px` | 2단 그리드, 사이드바(축소형) 또는 상단 메뉴 |
| **Desktop** | `≥ 1024px` | 3~4단 그리드, 고정 사이드바, 데이터 테이블 뷰 |

### 3.2. 구조 (Structure)

*   **PC/Tablet**: 좌측 고정 사이드바(250px) + 우측 컨텐츠 영역
*   **Mobile**: 상단 헤더(56px) + 컨텐츠 영역 + 하단 탭 바(60px)

## 4. 컴포넌트 가이드 (Component Guide)

### 4.1. 버튼 (Buttons)

*   **Primary**: 배경색 Primary/Secondary, 텍스트 White (주요 액션)
*   **Secondary**: 배경색 White, 테두리 Gray-300 (보조 액션, 닫기)
*   **Ghost**: 배경 투명, 호버 시 배경색 변경 (아이콘 버튼 등)
*   **Mobile**: 터치 영역 확보를 위해 높이 최소 **44px** 준수

### 4.2. 데이터 테이블 (Responsive Table)

*   **PC**: 일반적인 행/열 구조의 테이블. 헤더 고정, 페이지네이션 하단 배치.
*   **Mobile**: **카드 리스트(Card List)** 형태로 자동 변환.
    *   각 행(Row)이 하나의 카드 컴포넌트로 렌더링됨.
    *   주요 정보(이름, 상태 등)는 상단에 크게, 보조 정보는 하단에 작게 배치.
    *   가로 스크롤을 지양하고 세로 스크롤로 탐색.

### 4.3. 모달 및 드로어 (Modals & Drawers)

*   **PC**: 화면 중앙에 팝업되는 모달(Dialog). 배경 Dim 처리.
*   **Mobile**:
    *   상세 정보: 화면 하단에서 올라오는 **바텀 시트(Bottom Sheet)** 또는 **전체 화면(Full Screen)** 모달.
    *   메뉴: 좌측/우측에서 슬라이드되는 **드로어(Drawer)**.

## 5. 화면별 상세 명세 (Screen Specifications)

### 5.1. 공통 및 인증 (Common & Auth)

#### 5.1.1. 인트로 (Landing Page)
*   **URL**: `/`
*   **구성**:
    *   **Hero Section**: 서비스 슬로건, '무료 체험 시작' CTA 버튼, 대시보드 예시 이미지.
    *   **Features**: 주요 기능(출역, 계약, 정산) 소개 카드 (3열 그리드 -> 모바일 1열).
    *   **Pricing**: 요금제 비교 테이블.
*   **Mobile**: 햄버거 메뉴로 GNB 축소.

#### 5.1.2. 로그인 (Login)
*   **URL**: `/login`, `/login/hq`, `/login/select`
*   **구성**:
    *   **로그인 선택**: 본사 관리자 / 소셜 로그인(현장, 노무자) 분기.
    *   **소셜 로그인**: 카카오, 네이버, 구글, Apple 로그인 버튼 제공.
    *   **역할 선택**: 소셜 로그인 후 가입된 역할(현장소장/팀장/노무자) 선택 화면.

#### 5.1.3. 구독 신청 (Subscribe)
*   **URL**: `/subscribe`
*   **구성**:
    *   **Step 1**: 사업자번호 인증 (API 연동).
    *   **Step 2**: 요금제 선택 (Pro / Enterprise).
    *   **Step 3**: 결제 수단 등록 (카드/CMS).
    *   **Step 4**: 완료 및 계정 생성.

### 5.2. 본사 관리자 (HQ Admin)

*   **대시보드**: 전체 현장 현황, 금일 총 출역 인원, 실시간 이슈(지각/결근), 계약 체결률 차트.
*   **현장 관리**: 현장 목록(카드/테이블), 현장 개설 모달, 현장별 상세 정보(공정률, 예산).
*   **노무자 관리**: 전체 노무자 풀(Pool), 블랙리스트 관리, 자격증/이수증 파일 확인.
*   **계약 관리**: 전자근로계약서 발송/서명 현황, 미체결 리스트, 일괄 발송 기능.
*   **정산 관리**: 현장별 월별 노무비 정산 내역, 지급 요청 승인/반려, 명세서 출력.

### 5.3. 현장 관리자 (Site Manager)

*   **대시보드**: 해당 현장의 금일 출역 현황, 날씨 정보, 공지사항.
*   **출역 관리**: 일별 출역 명부, 안면인식/태그 기록, 공수 수정 및 마감 처리.
*   **작업 일보**: 금일 작업 내용(공종, 위치, 내용) 작성, 사진 첨부, 자재 투입 내역.
*   **팀/노무자**: 현장 투입 팀 관리, 신규 노무자 등록(QR/초대).

### 5.4. 노무자 (Worker)

*   **대시보드**: 나의 출역 현황(이번 달 공수/예상 급여), 현장 공지사항.
*   **출역 조회**: 일별 출퇴근 기록 및 인정 공수 확인 (캘린더 뷰).
*   **전자 계약**: 서명 요청된 계약서 목록, 모바일 서명(Canvas), 계약서 보관함.
*   **내 정보**: 프로필 사진, 계좌 정보, 자격증 업로드, 기초안전보건교육 이수증.

### 5.5. 슈퍼 어드민 (Super Admin)

*   **대시보드**: 전체 MRR(월 반복 매출), 가입 테넌트 수, 시스템 리소스 현황.
*   **고객사 관리**: 테넌트 목록, 상태 변경(정상/정지), 상세 정보 조회.
*   **구독/결제**: 결제 실패 내역, 재결제 처리, 세금계산서 발행 내역(홈택스 연동).

## 6. 모바일 최적화 가이드 (Mobile Optimization)

### 6.1. 내비게이션 (Navigation)
*   **PC**: 좌측 사이드바 메뉴.
*   **Mobile**:
    *   **하단 탭 바 (Bottom Tab Bar)**: 자주 쓰는 핵심 메뉴 4~5개 배치 (예: 홈, 출역, 계약, 더보기).
    *   **더보기 (More)**: 하단 탭에 없는 메뉴는 '더보기' 탭 또는 햄버거 메뉴로 접근.

### 6.2. 입력 폼 (Forms)
*   **Input**: 모바일 키보드 올라옴을 고려하여 입력 필드 터치 시 자동 스크롤.
*   **Select**: 네이티브 셀렉트(Native Select) 또는 바텀 시트(Bottom Sheet) 활용.
*   **Date Picker**: 모바일 친화적인 달력 UI 또는 네이티브 데이트 피커 사용.

### 6.3. 터치 인터랙션 (Touch Interaction)
*   **Swipe**: 리스트 항목 스와이프 시 삭제/수정 등 빠른 액션 제공 (선택 사항).
*   **Pull to Refresh**: 화면 당겨서 새로고침 기능 적용 (대시보드, 리스트).

---

## 7. 접근성 (Accessibility)

*   **색상 대비**: 텍스트와 배경 간 명도 대비 4.5:1 이상 준수.
*   **키보드 접근**: 모든 인터랙티브 요소(버튼, 링크, 입력창)는 키보드 탭(Tab)으로 이동 가능해야 함.
*   **대체 텍스트**: 이미지, 아이콘 버튼에는 적절한 `alt` 텍스트 또는 `aria-label` 제공.
