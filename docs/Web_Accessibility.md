# [SmartCON] 웹 접근성(Web Accessibility) 개발 가이드

> **개요:** 본 문서는 W3C WCAG 2.1/2.2 및 KWCAG 2.2 표준을 준수하여, 장애인 및 고령자를 포함한 모든 사용자가 '스마트콘' 서비스에 동등하게 접근할 수 있도록 하기 위한 기술 가이드라인입니다.

---

## 1. 인식의 용이성 (Perceivable)
*정보와 사용자 인터페이스 요소는 사용자가 인지할 수 있는 방식으로 표시되어야 한다.*

### 1.1 텍스트가 아닌 콘텐츠 (Text Alternatives)
* **지침:** `<img>`, 버튼, 아이콘 등 시각적 요소에는 반드시 그 의미나 용도를 알 수 있는 대체 텍스트를 제공한다.
* **구현 예시:**
    ```html
    <img src="btn_search.png"> 
    
    <img src="btn_search.png" alt="검색">
    <img src="deco_bg.png" alt="">
    ```

### 1.2 시간 의존 미디어 (Time-based Media)
* **지침:** 오디오나 비디오 콘텐츠에는 자막(Caption), 대본(Transcript), 또는 수어 통역을 제공해야 한다.

### 1.3 적응성 (Adaptable)
* **지침:** 콘텐츠는 정보나 구조의 손실 없이 다른 형태(예: 단순화된 레이아웃, 스크린 리더 순서)로 제공될 수 있어야 한다.
* **시멘틱 마크업:** `<div>` 남발 대신 `<header>`, `<nav>`, `<main>`, `<footer>`, `<h1>~<h6>`를 사용하여 문서 구조를 명확히 한다.

### 1.4 판별 가능성 (Distinguishable)
* **명도 대비:** 텍스트와 배경 간의 명도 대비는 최소 **4.5:1** 이상이어야 한다. (큰 텍스트는 3:1)
* **색상 무관:** 색상만으로 정보를 전달해서는 안 된다. (예: "붉은색 버튼을 누르세요" -> "붉은색 '삭제' 버튼을 누르세요")

## 2. 운용의 용이성 (Operable)
*사용자 인터페이스 구성요소와 내비게이션은 운용 가능해야 한다.*

### 2.1 키보드 접근성 (Keyboard Accessible)
* **지침:** 모든 기능은 마우스 없이 키보드(Tab, Enter 등)만으로 사용할 수 있어야 한다.
* **초점 이동:** 논리적인 순서(좌->우, 상->하)로 초점이 이동해야 하며, 초점(Focus Ring)은 시각적으로 명확해야 한다.

### 2.2 충분한 시간 제공 (Enough Time)
* **지침:** 시간 제한이 있는 기능(로그인 연장 등)은 사용자가 시간을 연장할 수 있는 수단을 제공해야 한다.

### 2.3 발작 예방 (Seizures and Physical Reactions)
* **지침:** 초당 3회 이상 번쩍이는 콘텐츠는 포함하지 않는다. (광과민성 발작 예방)

### 2.4 탐색 용이성 (Navigable)
* **지침:** 반복되는 영역(헤더, 사이드바)을 건너뛸 수 있는 **'본문 바로가기(Skip Navigation)'** 링크를 제공한다.
* **페이지 제목:** `<title>` 태그에 페이지의 고유한 제목을 명시한다.

---

## 3. 이해의 용이성 (Understandable)
*정보와 사용자 인터페이스 운용은 이해할 수 있어야 한다.*

### 3.1 가독성 (Readable)
* **지침:** `<html>` 태그에 `lang` 속성을 명시하여 스크린 리더가 정확한 언어로 읽도록 한다.
    ```html
    <html lang="ko">
    ```

### 3.2 예측 가능성 (Predictable)
* **지침:** 사용자 인터페이스의 컴포넌트(입력창, 버튼 등)는 초점을 받거나 입력할 때, 사용자가 예측하지 못한 맥락 변경(새 창 열림, 자동 제출 등)이 일어나지 않아야 한다.

### 3.3 입력 지원 (Input Assistance)
* **지침:** 입력 오류가 발생하면, 오류가 난 항목을 텍스트로 설명하고 수정 방법을 제안해야 한다.
* **레이블 제공:** `<input>` 요소에는 반드시 대응하는 `<label>`을 제공하거나 `title`, `aria-label` 속성을 사용한다.
    ```html
    <label for="user_id">아이디</label>
    <input type="text" id="user_id">
    ```

---

## 4. 견고성 (Robust)
*콘텐츠는 보조 기술을 포함한 다양한 사용자 에이전트가 해석할 수 있도록 견고해야 한다.*

### 4.1 호환성 (Compatible)
* **지침:** 웹 표준(HTML/CSS) 문법을 준수한다. (태그 열고 닫기, 속성 중복 방지, ID 유니크함 유지)
* **WAI-ARIA:** 표준 HTML 태그로 표현할 수 없는 동적 UI(탭, 모달, 아코디언 등)에는 적절한 ARIA 속성(`role`, `aria-expanded`, `aria-hidden` 등)을 사용한다.

## 5. [SmartCON] 개발 체크리스트

- [ ] 이미지 요소에 `alt` 속성이 적절히 제공되었는가?
- [ ] 마우스 없이 키보드(`Tab`)만으로 모든 메뉴와 버튼 이용이 가능한가?
- [ ] 텍스트 명도 대비가 4.5:1 이상인가?
- [ ] 폼 요소(Input)에 레이블이 연결되어 있는가?
- [ ] HTML 문법 오류(태그 미닫힘, ID 중복)가 없는가?
- [ ] 모바일 환경에서 터치 영역이 충분한가? (최소 44x44px 권장)
