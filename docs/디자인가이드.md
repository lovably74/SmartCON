## 1. 개요 (Overview)

본 문서는 건설 현장 인력 관리 SaaS 플랫폼 **'스마트콘 라이트(SmartCON Lite)'**의 사용자 인터페이스(UI) 디자인 및 구현을 위한 가이드라인입니다. 본사 관리자, 현장 관리자, 노무자, 슈퍼 어드민 등 다양한 사용자 그룹에게 일관된 경험을 제공하고, PC부터 모바일까지 모든 디바이스에서 최적화된 사용성을 보장하는 것을 목표로 합니다.

### 1.1 디자인 철학
- **신뢰성**: 건설 현장의 전문성을 반영하는 안정적이고 신뢰할 수 있는 디자인
- **직관성**: 복잡한 기능을 단순하고 직관적으로 표현
- **접근성**: 모든 사용자가 쉽게 접근할 수 있는 포용적 디자인
- **효율성**: 업무 효율성을 극대화하는 사용자 중심 인터페이스

### 1.2 핵심 가치
- **현장 중심**: 건설 현장의 실제 업무 환경을 고려한 실용적 디자인
- **모바일 우선**: 현장에서 주로 사용되는 모바일 환경 최적화
- **역할별 최적화**: 5단계 사용자 역할별 맞춤형 인터페이스
- **일관성**: 모든 화면과 기능에서 일관된 디자인 언어 적용

## 2. 디자인 시스템 (Design System)

### 2.1. 컬러 팔레트 (Color Palette)

신뢰감과 전문성을 전달하는 **Dark Navy**를 주조색으로, 명확한 인지와 행동 유도를 위한 **Teal**과 **Blue**를 보조색으로 사용합니다.

#### 2.1.1 주요 색상 (Primary Colors)
| 구분 | 색상명 | Hex Code | RGB | 용도 |
| :--- | :--- | :--- | :--- | :--- |
| **Primary** | **Dark Navy** | `#0F172A` | rgb(15, 23, 42) | 헤더, 사이드바, 주요 텍스트, 브랜드 아이덴티티 |
| **Primary Light** | **Navy Blue** | `#1E293B` | rgb(30, 41, 59) | 호버 상태, 활성 메뉴 |
| **Secondary** | **Teal Green** | `#10B981` | rgb(16, 185, 129) | 성공 상태, 강조 버튼, 긍정적 지표 (출역 마감 등) |
| **Secondary Light** | **Light Teal** | `#34D399` | rgb(52, 211, 153) | 호버 상태, 보조 강조 |
| **Accent** | **Sky Blue** | `#0EA5E9` | rgb(14, 165, 233) | 링크, 활성 상태, 정보성 알림 |
| **Accent Light** | **Light Blue** | `#38BDF8` | rgb(56, 189, 248) | 호버 상태, 보조 정보 |

#### 2.1.2 시스템 색상 (System Colors)
| 구분 | 색상명 | Hex Code | RGB | 용도 |
| :--- | :--- | :--- | :--- | :--- |
| **Success** | **Green** | `#22C55E` | rgb(34, 197, 94) | 성공 메시지, 완료 상태 |
| **Warning** | **Orange** | `#F97316` | rgb(249, 115, 22) | 경고, 대기 상태 (지각, 서명 대기) |
| **Error** | **Red** | `#EF4444` | rgb(239, 68, 68) | 오류, 삭제, 위험 상태 (결근, 계약 만료 등) |
| **Info** | **Blue** | `#3B82F6` | rgb(59, 130, 246) | 정보성 알림, 도움말 |

#### 2.1.3 중성 색상 (Neutral Colors)
| 구분 | 색상명 | Hex Code | RGB | 용도 |
| :--- | :--- | :--- | :--- | :--- |
| **Background** | **Light Gray** | `#F8FAFC` | rgb(248, 250, 252) | 전체 배경색 (눈의 피로도 감소) |
| **Surface** | **White** | `#FFFFFF` | rgb(255, 255, 255) | 카드, 모달, 컨텐츠 영역 배경 |
| **Border** | **Gray 200** | `#E5E7EB` | rgb(229, 231, 235) | 테두리, 구분선 |
| **Text Primary** | **Gray 900** | `#111827` | rgb(17, 24, 39) | 주요 텍스트 |
| **Text Secondary** | **Gray 600** | `#4B5563` | rgb(75, 85, 99) | 보조 텍스트 |
| **Text Muted** | **Gray 400** | `#9CA3AF` | rgb(156, 163, 175) | 비활성 텍스트, 플레이스홀더 |

#### 2.1.4 색상 사용 원칙
- **배경(큰 면적)**: `#FAFAFA`(Off-White) 기반의 중립 배경을 기본값으로 사용
- **Primary 색상 제한**: 핵심 CTA/활성 상태/중요 강조(포인트) 중심으로만 사용
- **대면적 채움 금지**: 강한 색(노랑/형광/순색) 및 Primary 풀배경은 눈부심 유발 → 금지
- **텍스트 대비**: 본문은 다크 그레이 계열(예: `#262626`), 보조 텍스트는 중간 회색(예: `#737373`)로 가독성 확보

### 2.2. 타이포그래피 (Typography)

가독성이 뛰어나고 현대적인 **Pretendard** 폰트를 기본으로 사용합니다.

#### 2.2.1 폰트 패밀리
```css
font-family: 'Pretendard', -apple-system, BlinkMacSystemFont, system-ui, Roboto, 'Helvetica Neue', 'Segoe UI', sans-serif;
```

#### 2.2.2 타이포그래피 스케일
| 구분 | 크기 (Desktop) | 크기 (Mobile) | 폰트 굵기 | 줄 높이 | 용도 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **H1 (Page Title)** | 32px | 24px | 700 (Bold) | 1.2 | 페이지 제목 |
| **H2 (Section Title)** | 24px | 20px | 600 (SemiBold) | 1.3 | 섹션 제목 |
| **H3 (Card Title)** | 20px | 18px | 600 (SemiBold) | 1.4 | 카드 제목, 서브섹션 |
| **H4 (Subsection)** | 18px | 16px | 600 (SemiBold) | 1.4 | 하위 섹션 |
| **Body Large** | 16px | 16px | 400 (Regular) | 1.5 | 중요한 본문 |
| **Body (Default)** | 14px | 14px | 400 (Regular) | 1.5 | 기본 본문 |
| **Body Small** | 12px | 12px | 400 (Regular) | 1.4 | 작은 본문, 라벨 |
| **Caption** | 11px | 11px | 400 (Regular) | 1.3 | 캡션, 메타 정보 |

#### 2.2.3 타이포그래피 사용 원칙
- **계층 구조**: 명확한 정보 계층을 위해 크기와 굵기를 체계적으로 사용
- **가독성**: 최소 12px 이상의 크기 사용, 충분한 줄 간격 확보
- **일관성**: 동일한 레벨의 정보는 동일한 타이포그래피 스타일 적용
- **접근성**: 색상에만 의존하지 않고 크기와 굵기로도 중요도 표현

### 2.3. 아이콘 (Iconography)

#### 2.3.1 아이콘 라이브러리
- **Library**: `Lucide React`
- **Style**: Stroke width 2px, Rounded corners
- **Alternative**: `Heroicons`, `Tabler Icons`

#### 2.3.2 아이콘 크기 가이드
| 용도 | 크기 | 사용 예시 |
| :--- | :--- | :--- |
| **Small** | 16px | 버튼 내 아이콘, 인라인 아이콘 |
| **Medium** | 20px | 메뉴 아이콘, 카드 아이콘 |
| **Large** | 24px | 헤더 아이콘, 주요 액션 |
| **XLarge** | 32px | 대시보드 위젯, 상태 표시 |
| **Illustration** | 48px+ | 빈 상태, 일러스트레이션 |

#### 2.3.3 아이콘 사용 원칙
- **의미 명확성**: 직관적으로 이해할 수 있는 아이콘 선택
- **일관성**: 동일한 의미는 동일한 아이콘 사용
- **접근성**: 아이콘만으로 의미 전달하지 않고 텍스트 라벨 병행
- **시각적 균형**: 텍스트와 아이콘의 시각적 균형 고려

## 3. 레이아웃 및 그리드 (Layout & Grid)

### 3.1. 반응형 브레이크포인트 (Breakpoints)

Tailwind CSS 표준을 따르며, 모바일 우선(Mobile First) 전략을 적용합니다.

| 디바이스 | Breakpoint | 컨테이너 너비 | 레이아웃 특징 |
| :--- | :--- | :--- | :--- |
| **Mobile** | `< 640px` | 100% | 1단 그리드, 하단 탭 바, 햄버거 메뉴, 카드 리스트 뷰 |
| **Tablet** | `640px ~ 1024px` | 640px | 2단 그리드, 사이드바(축소형) 또는 상단 메뉴 |
| **Desktop** | `1024px ~ 1280px` | 1024px | 3~4단 그리드, 고정 사이드바, 데이터 테이블 뷰 |
| **Large Desktop** | `≥ 1280px` | 1280px | 4단 그리드, 확장된 사이드바 |

### 3.2. 그리드 시스템

#### 3.2.1 12컬럼 그리드 시스템
```css
.grid-container {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 24px;
}

/* 반응형 그리드 */
@media (max-width: 768px) {
  .grid-container {
    grid-template-columns: 1fr;
    gap: 16px;
  }
}
```

#### 3.2.2 레이아웃 구조
**PC/Tablet 레이아웃:**
```
┌─────────────────────────────────────┐
│ Header (고정, 64px)                  │
├─────────┬───────────────────────────┤
│ Sidebar │ Main Content              │
│ (250px) │                           │
│         │                           │
│         │                           │
└─────────┴───────────────────────────┘
```

**Mobile 레이아웃:**
```
┌─────────────────────────────────────┐
│ Header (고정, 56px)                  │
├─────────────────────────────────────┤
│                                     │
│ Main Content                        │
│                                     │
│                                     │
├─────────────────────────────────────┤
│ Bottom Navigation (60px)            │
└─────────────────────────────────────┘
```

### 3.3. 컨테이너 및 여백

#### 3.3.1 컨테이너 설정
```css
.container {
  max-width: 1280px;
  margin: 0 auto;
  padding: 0 24px;
}

@media (max-width: 768px) {
  .container {
    padding: 0 16px;
  }
}
```

#### 3.3.2 안전 영역 (Safe Area)
- **모바일**: 상단 노치, 하단 홈 인디케이터 고려
- **데스크톱**: 최소 24px 여백 확보
- **터치 영역**: 최소 44px × 44px 확보

### 4.4. 데이터 테이블 (Data Tables)

#### 4.4.1 반응형 테이블 전략
**PC 환경:**
- 일반적인 행/열 구조의 테이블
- 헤더 고정, 페이지네이션 하단 배치
- 정렬, 필터링 기능 제공

**모바일 환경:**
- **카드 리스트(Card List)** 형태로 자동 변환
- 각 행(Row)이 하나의 카드 컴포넌트로 렌더링
- 주요 정보(이름, 상태 등)는 상단에 크게, 보조 정보는 하단에 작게 배치
- 가로 스크롤을 지양하고 세로 스크롤로 탐색

#### 4.4.2 테이블 스타일
```css
/* PC 테이블 */
.table {

  border-collapse: collapse;
  background: #FFFFFF;

  padding: 12px 16px;
  text-align: left;

  border-bottom: 1px solid #E5E7EB;
}

.table td {
  padding: 12px 16px;

/* 모바일 카드 리스트 */
@media (max-width: 768px) {
  .table-responsive {
    display: none;
  }
  
  .card-list {
    display: block;
  }
}
```

### 4.5. 모달 및 드로어 (Modals & Drawers)

#### 4.5.1 모달 사용 가이드
**PC 환경:**
- 화면 중앙에 팝업되는 모달(Dialog)
- 배경 Dim 처리 (rgba(0, 0, 0, 0.5))
- ESC 키로 닫기 가능

// 시스템 다크 모드 감지
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)');

// 테마 변경 함수
const setTheme = (theme) => {

  localStorage.setItem('theme', theme);

// 초기 테마 설정
const savedTheme = localStorage.getItem('theme');
const initialTheme = savedTheme || (prefersDark.matches ? 'dark' : 'light');
setTheme(initialTheme);
