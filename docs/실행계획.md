## 2. 재정비된 개발 단계

### Phase 0: 프로토타입 완성 (2주) 🎯 **최우선**
**목표**: 완전한 기능을 가진 프로토타입으로 사용자 피드백 수집

#### 0.1. 프론트엔드 프로토타입 완성 (1주)
- [ ] **인트로 페이지**: ismartcon.net과 동일한 구성으로 구현
- [ ] **인증 플로우**: 로그인 방식 선택 → HQ/소셜 로그인 → 역할 선택
- [ ] **역할별 대시보드**: 5개 역할(Super/HQ/Site/Team/Worker) 대시보드
- [ ] **핵심 화면 구현**:
  - HQ: 현장관리, 노무자관리, 계약관리, 정산관리, 설정
  - Site: 출역관리, 작업일보, 계약관리, 팀관리, 급여관리
  - Worker: 출역조회, 계약서, 내정보
- [ ] **반응형 디자인**: PC/태블릿/모바일 완전 대응
- [ ] **모든 팝업/모달**: 등록, 수정, 삭제, 상세보기 모달 구현
- [ ] **헤더 기능**: 다국어, 현장변경, 알림 아이콘 및 패널

#### 0.2. 목업 데이터 및 상태관리 (3일)
- [ ] **완전한 목업 데이터**: 실제 데이터처럼 보이는 완전한 샘플 데이터
- [ ] **Zustand 상태관리**: 사용자, 현장, 출역 등 상태 관리
- [ ] **라우팅 완성**: 모든 화면 간 네비게이션 구현
- [ ] **에러 처리**: 404, 권한 없음 등 에러 페이지

#### 0.3. 사용자 테스트 준비 (2일)
- [ ] **배포 환경**: Vercel/Netlify 등 정적 호스팅 배포
- [ ] **테스트 시나리오**: 각 역할별 사용자 테스트 시나리오 작성
- [ ] **피드백 수집**: 피드백 수집 방법 및 도구 준비

### Phase 1: 핵심 인프라 및 인증 (3주)
**목표**: 프로덕션 기반 인프라 및 인증 시스템 구축

#### 1.1. 개발 환경 및 인프라 (1주)
- [ ] **Docker 환경**: MariaDB, Redis, MinIO 컨테이너 구성
- [ ] **CI/CD 파이프라인**: GitHub Actions 기본 워크플로우
- [ ] **환경 설정**: 개발/스테이징/프로덕션 환경 분리
- [ ] **로깅 시스템**: 구조화된 로깅 및 모니터링 설정

#### 1.2. 백엔드 핵심 모듈 (1주)
- [ ] **멀티테넌트 구현**: Hibernate Filter 기반 테넌트 격리
- [ ] **공통 모듈**: ApiResponse, GlobalExceptionHandler, 감사로그
- [ ] **보안 설정**: Spring Security + JWT 구현
- [ ] **데이터베이스**: 핵심 엔티티 및 Repository 구현

#### 1.3. 인증 시스템 (1주)
- [ ] **JWT 인증**: 토큰 생성, 검증, 갱신 로직
- [ ] **소셜 로그인**: 카카오, 네이버 OAuth2 연동
- [ ] **권한 관리**: 역할 기반 접근 제어 (RBAC)
- [ ] **API 보안**: 모든 API 엔드포인트 보안 적용

### Phase 2: 핵심 비즈니스 로직 (4주)
**목표**: 출역관리, 현장관리 등 핵심 비즈니스 기능 구현

#### 2.1. 현장 및 사용자 관리 (1주)
- [ ] **현장 관리**: CRUD, 관리자 배정, 설정 관리
- [ ] **사용자 관리**: 초대, 역할 부여, 프로필 관리
- [ ] **팀 관리**: 팀 생성, 팀원 관리, 권한 설정
- [ ] **테스트 코드**: 단위 테스트 및 통합 테스트 작성

#### 2.2. 작업 프로세스 (1주)
- [ ] **작업 요청**: Site → Team 작업 요청 시스템
- [ ] **작업 수락**: Team의 작업 수락/거부 처리
- [ ] **인원 배정**: 작업별 인원 배정 및 관리
- [ ] **알림 시스템**: Push 알림 및 SMS 발송

#### 2.3. 출역 관리 기본 (1주)
- [ ] **출역 기록**: 수동 출역 체크 및 기록 관리
- [ ] **출역 조회**: 일별/월별 출역 현황 조회
- [ ] **출역 수정**: 이의제기 및 수정 처리
- [ ] **통계 집계**: 출역 통계 및 리포트 생성

#### 2.4. 공사일보 시스템 (1주)
- [ ] **날씨 API**: OpenWeatherMap 연동
- [ ] **일보 작성**: 팀장의 일보 작성 및 제출
- [ ] **일보 승인**: 현장관리자의 일보 승인 처리
- [ ] **PDF 생성**: 일보 PDF 출력 기능

### Phase 3: 안면인식 및 고급 기능 (4주)
**목표**: 안면인식 시스템 및 전자계약 등 고급 기능 구현

#### 3.1. 안면인식 파이프라인 (2주)
- [ ] **안면 등록**: 사진 업로드 및 전처리
- [ ] **FaceNet 연동**: 임베딩 추출 및 벡터 저장
- [ ] **동기화 배치**: 일일 활성 리스트 동기화
- [ ] **인식 로그**: 안면인식 결과 수집 및 처리

#### 3.2. 전자계약 시스템 (1주)
- [ ] **계약서 템플릿**: PDF 템플릿 및 데이터 바인딩
- [ ] **전자서명**: Canvas 기반 서명 및 저장
- [ ] **계약 관리**: 계약 상태 관리 및 알림
- [ ] **보안 처리**: 계약서 해시 및 위변조 방지

#### 3.3. 정산 및 세무 (1주)
- [ ] **노무비 계산**: 공수 기반 노무비 자동 계산
- [ ] **급여 명세서**: PDF 급여 명세서 생성
- [ ] **세금계산서**: 홈택스 API 연동
- [ ] **결제 연동**: PG사 연동 및 자동 결제

### Phase 4: 모바일 앱 및 최적화 (3주)
**목표**: 모바일 네이티브 앱 및 성능 최적화

2. **팝업/모달 시스템**: 등록, 수정, 삭제, 상세보기 모달
3. **상태 관리 완성**: Zustand 기반 전역 상태 관리
4. **사용자 테스트 준비**: 배포 및 피드백 수집 준비
