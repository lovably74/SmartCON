# 건설현장 인력관리 SaaS 서비스 기능 정의서

## 1. 개요
본 문서는 건설현장 인력관리 SaaS 서비스의 개발을 위한 상세 기능 정의서입니다. 본 서비스는 건설 현장의 복잡한 인력 관리 프로세스를 디지털화하여 업무 효율성을 극대화하고, 투명하고 정확한 노무 관리를 실현하는 것을 목표로 합니다.

## 2. 사용자 역할 정의
서비스는 다음과 같은 4가지 핵심 사용자 역할을 정의합니다.

| 역할 | 설명 | 주요 책임 |
| :--- | :--- | :--- |
| **본사 관리자 (HQ Admin)** | 건설사 본사의 관리자 | 전사적 현장/사용자 관리, 표준 정보 관리, 전체 현황 모니터링 |
| **현장 관리자 (Site Manager)** | 개별 건설 현장의 관리 소장 또는 공무 담당자 | 현장 개설/운영, 작업 지시, 출역 승인, 노무비 정산 |
| **노무 팀장 (Team Leader)** | 현장에서 작업을 수행하는 팀(반)의 리더 | 팀원 관리, 작업 수락, 출역 정보 관리, 팀원 근태 관리 |
| **노무자 (Worker)** | 실제 현장에서 근로를 제공하는 작업자 | 개인 출역 내역 확인, 근로계약 체결, 개인정보 관리 |

## 3. 상세 기능 명세

### 3.1. 본사 관리자 (HQ Admin)
전사적인 관점에서 시스템의 기준 정보를 설정하고, 전체 현장의 운영 현황을 감독합니다.

| 기능 그룹 | 상세 기능 | 주요 명세 |
| :--- | :--- | :--- |
| **현장 관리** | 현장 등록/수정 | 현장명, 주소, 공사기간, 공사금액 등 기본 정보 관리 |
| | 현장 목록 조회 | 전체 현장의 상태(진행중, 종료 등) 및 주요 지표 조회 |
| **사용자 관리** | 본사 관리자 관리 | 본사 관리자 계정 생성, 수정, 권한 부여 및 삭제 |
| | 현장 관리자 관리 | 각 현장의 관리자 계정 생성 및 현장 매핑 |
| **기준 정보 관리** | 공종 코드 관리 | 표준 공종 코드 체계 수립 및 관리 (대/중/소분류) |
| | 직종 코드 관리 | 표준 직종 코드 체계 수립 및 관리 |
| **대시보드** | 전사 현황 모니터링 | 전체 현장 수, 총 출력 인원, 금일 출력 현황 등 핵심 지표 시각화 |

### 3.2. 현장 관리자 (Site Manager)
개별 현장의 운영을 책임지며, 실질적인 인력 투입 계획과 실행을 관리합니다.

| 기능 그룹 | 상세 기능 | 주요 명세 |
| :--- | :--- | :--- |
| **현장 설정** | 공종/직종 설정 | 본사 표준 코드 중 해당 현장에서 사용할 코드 선택 및 단가 설정 |
| | 현장 정보 관리 | 현장 세부 정보 및 운영 규칙 설정 |
| **팀 관리** | 팀 등록/초대 | 작업 팀 등록 및 팀장 초대 (SMS/알림톡 발송) |
| | 팀 구성 승인 | 팀장이 등록한 팀원 명단 검토 및 승인 |
| **작업 관리** | 작업 지시 (배정) | 날짜, 공종, 필요 인원, 작업 내용을 포함하여 특정 팀에게 작업 요청 |
| | 작업 현황 조회 | 팀별 작업 수락 여부 및 진행 상태 모니터링 |
| **출역 관리** | 일일 출역 마감 | 당일 출역 인원 확인, 공수 조정 및 최종 승인 (마감 처리) |
| | 출역 현황 조회 | 일별/월별/팀별/공종별 상세 출역 내역 조회 및 엑셀 다운로드 |
| **정산 관리** | 노무비 명세서 | 기간별, 팀별 노무비 산출 내역 생성 및 조회 |
| | 노무비 청구서 | 노무비 지급을 위한 청구서 양식 생성 및 출력 |
| **대시보드** | 현장 현황 모니터링 | 금일 출역률, 공종별 투입 현황, 날씨 정보 등 현장 운영 지표 제공 |

### 3.3. 노무 팀장 (Team Leader)
자신의 팀원을 관리하고, 현장 관리자의 작업 지시를 이행하며 팀원의 근태를 관리합니다.

| 기능 그룹 | 상세 기능 | 주요 명세 |
| :--- | :--- | :--- |
| **팀원 관리** | 팀원 등록/초대 | 이름, 연락처 등으로 팀원 초대 및 등록 관리 |
| | 팀원 정보 관리 | 팀원의 기본 정보 및 자격 사항 관리 |
| **작업 관리** | 작업 수락/거부 | 현장 관리자가 배정한 작업 요청 확인 및 수락/거부 처리 |
| | 작업 팀원 배정 | 수락한 작업에 투입할 팀원 명단 확정 및 전송 |
| **출역 관리** | 출역 정보 입력 | 팀원의 당일 출역 여부, 공수, 특이사항 입력 및 제출 |
| | 이의 제기 | 현장 관리자의 출역 승인 내역에 대한 수정 요청 (이의 제기) |
| **대시보드** | 팀 운영 현황 | 배정된 작업 목록, 금일 출역 현황, 팀원 근태 현황 요약 |

### 3.4. 노무자 (Worker)
자신의 근로 내역을 투명하게 확인하고, 근로 계약 등 필수 행정 절차를 간편하게 수행합니다.

| 기능 그룹 | 상세 기능 | 주요 명세 |
| :--- | :--- | :--- |
| **내 정보** | 출역 내역 조회 | 일별/월별 자신의 출역 기록(공수, 단가, 예상 급여) 상세 조회 |
| | 근로계약 관리 | 전자 근로계약서 조회 및 서명 (전자서명 기능) |
| **대시보드** | 개인 현황 | 월간 누적 공수, 예상 급여액, 최근 출역 현장 등 개인화 정보 제공 |

## 4. 공통 기능 요구사항
모든 사용자에게 공통적으로 제공되는 기반 기능입니다.

*   **알림 (Notifications)**: 작업 배정, 승인, 공지사항 등 주요 이벤트 발생 시 Push/SMS/알림톡 발송
*   **내 정보 관리 (Profile)**: 개인 정보 수정, 비밀번호 변경, 자격증 사본 업로드 등
*   **공지사항 (Notice)**: 전체 공지 및 현장별 공지사항 조회 (권한에 따른 작성 기능 포함)
*   **안면인식 연동**: 현장 출퇴근 기록을 위한 안면인식 단말기/시스템 연동 (출역 데이터 자동 수집)
*   **로그인 및 접근 제어**:
    *   **본사**: ID/PW 방식 로그인
    *   **현장(모바일)**: 소셜 로그인 (카카오, 네이버 등) 및 간편 인증 지원
    *   **다중 소속 지원**: 한 사용자가 여러 현장 또는 여러 역할(팀장 겸 노무자 등)을 수행할 경우 컨텍스트 전환 기능 제공

## 5. 비기능 요구사항 (System Architecture)
*   **SaaS 아키텍처**: 멀티 테넌트(Multi-tenant) 구조로 데이터 격리 및 보안성 확보
*   **모듈식 설계**: 코어 기능과 확장 기능을 분리하여 유연한 기능 추가/삭제 지원
*   **반응형 웹 (Responsive Web)**: PC, 태블릿, 모바일 등 다양한 디바이스 환경 지원 (Mobile First 권장)
*   **기술 스택**:
    *   Frontend: React.js 또는 Vue.js (SPA 구조)
    *   Backend: Node.js, Python, Java 등 (MSA 또는 모듈러 모놀리식 고려)
    *   Database: PostgreSQL, MySQL 등 오픈소스 RDBMS 권장
*   **UX/UI**: 직관적이고 사용하기 쉬운 인터페이스 제공 (별도 매뉴얼 없이 사용 가능하도록 설계)
*   **딥링크 (Deep Linking)**: 초대 메시지 등을 통해 앱 설치 유도 및 특정 화면으로의 직접 이동 지원
*   **다국어 지원**: 한국어(기본), 영어 지원 (향후 확장 가능성 고려)
