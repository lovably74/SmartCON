건설현장 인력관리 SaaS 서비스 기획서


--------------------------------------------------------------------------------


1. 서비스 개요 (Service Overview)

본 서비스 기획서는 건설현장 인력관리 SaaS(Software as a Service) 플랫폼 개발을 위한 기능 명세와 요구사항을 정의하는 것을 목적으로 합니다. 전통적인 건설 현장의 인력 관리는 파편화된 정보, 수기 중심의 비효율적 프로세스, 그리고 이해관계자 간의 소통 부재 등 복잡한 문제에 직면해 있습니다. 본 서비스는 이러한 문제점을 해결하기 위해 데이터 중앙화, 핵심 프로세스 자동화, 그리고 모든 참여자 간의 투명한 소통 채널 제공을 핵심 전략으로 삼습니다. 이를 통해 건설사는 운영 효율성을 극대화하고, 현장 관리자는 실무 부담을 경감하며, 근로자는 자신의 권익을 보호받는 선순환 생태계를 구축하고자 합니다.

본 서비스가 제공하는 핵심 가치 제안(Core Value Proposition)은 다음과 같습니다.

* 데이터 기반 통합 관리 및 신뢰 구축: 파편화된 현장 정보를 단일 플랫폼에 통합하여 실시간 전사 현황 파악을 지원합니다. 모든 출역 및 계약 정보를 이해관계자에게 투명하게 공개함으로써, 건설사와 근로자 간의 고질적인 불신 문제를 해소하고 상호 신뢰 관계를 구축합니다.
* 업무 프로세스 자동화를 통한 생산성 향상: 출역 관리, 공사일보 작성, 근로계약서 관리 등 반복적이고 시간이 많이 소요되는 작업을 자동화하여 현장 관리자와 본사 관리자의 업무 생산성을 획기적으로 향상시킵니다.
* 역할 기반 맞춤 기능과 실시간 소통 강화: 본사 관리자부터 현장 노무자에 이르기까지 각 사용자 역할에 최적화된 맞춤형 기능과 대시보드를 제공하고, 실시간 알림 시스템을 통해 중요 정보를 즉시 공유하여 원활한 협업을 지원합니다.

이어지는 섹션에서는 서비스의 성공을 좌우하는 사용자 중심 설계를 위해 각 사용자 역할을 명확히 정의할 것입니다. 이는 모든 기능 명세의 논리적 기반이 될 것입니다.

2. 사용자 역할 및 정의 (User Roles and Definitions)

성공적인 SaaS 서비스는 사용자에 대한 깊은 이해에서 출발합니다. 사용자 중심의 기능 설계를 위해 각 사용자 역할(Persona)을 명확히 정의하는 것은 필수적입니다. 건설 현장 생태계는 본사, 현장, 노무팀, 그리고 개별 노무자로 이어지는 명확한 위계와 협업 구조를 가집니다. 각 역할이 갖는 고유한 책임과 목표를 명확히 함으로써, 서비스는 각 사용자에게 꼭 필요한 가치를 제공하고 전반적인 시스템 도입 성공률을 높일 수 있습니다.

본 서비스는 다음과 같은 네 가지 핵심 사용자 역할로 구분되며, 각 역할의 책임과 시스템 사용 목표는 아래와 같습니다.

* 본사 관리자 (HQ Administrator)
  * 핵심 책임: 전사적 차원에서 모든 건설 현장의 운영 현황과 인력 데이터를 총괄 감독합니다. 회사의 기본 정보, 공종, 장비, 노무 단가 등 기준 정보를 설정하고 관리합니다.
  * 시스템 사용 목표: 여러 현장의 데이터를 종합적으로 분석하여 비즈니스 의사결정을 지원하고, 일관된 회사 정책을 모든 현장에 적용하며, 전사적 리스크를 관리합니다.
* 현장 관리자 (Site Manager)
  * 핵심 책임: 담당하는 개별 건설 현장의 일상적인 운영을 총괄합니다. 노무팀 배정, 출역 현황 관리, 공사일보 작성 및 승인, 근로계약 관리 등 현장 실무를 책임집니다.
  * 시스템 사용 목표: 실시간 데이터에 기반하여 현장 자원을 효율적으로 배분하고, 발생 가능한 이슈에 신속하게 대응하며, 현장 운영의 생산성과 안전성을 극대화합니다.
* 노무팀장 (Labor Team Lead)
  * 핵심 책임: 소속된 노무팀원을 관리하고 현장에 필요한 인력을 공급합니다. 현장 관리자와의 작업 일정 및 내용을 조율하고, 팀원의 출역 현황을 취합하여 보고하며, 팀원의 권익을 대변합니다.
  * 시스템 사용 목표: 팀원 구성 및 관리를 용이하게 하고, 현장과의 커뮤니케이션 오류를 최소화하며, 팀원의 출역 기록을 정확하게 관리하고 필요한 경우 이의를 제기합니다.
* 노무자 (Worker/Laborer)
  * 핵심 책임: 배정된 작업 구간에서 실제 건설 업무를 수행하는 주체입니다. 자신의 근로 조건을 확인하고 준수할 의무가 있습니다.
  * 시스템 사용 목표: 자신의 출역 기록, 공수, 예상 노임 등 근로 정보를 투명하게 확인하고, 근로계약서에 전자 서명하며, 개인 정보를 직접 관리하여 신뢰도 높은 근로 환경을 보장받습니다.

이처럼 명확하게 정의된 사용자 역할은 이어지는 상세 기능 명세의 논리적 기반이 되어, 각 사용자에게 최적화된 경험을 제공하는 서비스 설계의 기틀을 마련합니다.

3. 사용자 유형별 상세 기능 명세 (Detailed Feature Specification by User Type)

본 섹션은 서비스의 핵심 기능들을 구체화하는 부분입니다. 앞서 정의한 각 사용자 역할이 부여받은 권한과 책임에 따라 기능이 어떻게 맞춤 설계되었는지 전략적으로 접근합니다. 각 기능은 사용자의 핵심 목표 달성을 지원하고, 불필요한 복잡성은 제거하여 직관적인 사용성을 제공하는 데 초점을 맞춥니다.

3.1. 본사 관리자 (HQ Administrator)

본사 관리자는 전사적 자원을 통제하고 비즈니스 현황을 종합적으로 파악하는 최고 관리자입니다. 제공되는 기능들은 여러 현장을 아우르는 데이터 조회, 기준 정보 설정, 그리고 상위 레벨의 관리 및 감독에 중점을 둡니다.

기능	주요 명세
회사정보 관리	회사의 기본 정보를 등록하고 수정하는 기능. <br> * 관리 필드: 회사명, 회사주소, 회사 대표번호, 사업자등록번호, 대표이사, 대표 이메일
현장 개설	신규 건설 현장을 시스템에 등록하는 기능. <br> * 입력 필드: 현장명, 안면인식기 설치여부, 안면인식기 고유번호, 현장주소, 현장 연락처, 현장소장 정보, 관리자 정보 <br> * 선택 필드: 참여공종 (본사 공종 리스트에서 중복 선택 가능)
현장 정보 조회	시스템에 등록된 모든 현장의 기본 정보를 조회하는 기능.
현장관리자 등록 (초대)	특정 현장을 관리할 현장관리자를 시스템에 등록하거나 초대하는 기능. <br> * 프로세스: <br> 1. 기등록된 현장관리자 목록에서 선택 <br> 2. 신규 등록 시, 이름, 주민번호, 핸드폰번호, 이메일 정보 입력 후 초대 (초대 시 SMS/알림톡/이메일로 링크 전송)
근로자 정보 조회	시스템에 등록된 모든 근로자의 정보를 조회하는 기능. <br> * 목록 조회 필드: 이름, 핸드폰번호, 이메일, 마지막 투입 현장(참여공종 포함) <br> * 상세 조회: 기본 상세정보 및 전체 투입 현장 이력(참여공종 포함) 목록
노무팀장 정보 조회	시스템에 등록된 모든 노무팀장의 정보를 조회하는 기능. <br> * 목록 조회 필드: 이름, 핸드폰번호, 이메일, 마지막 투입 현장(참여공종 포함) <br> * 상세 조회: 기본 상세정보 및 전체 투입 현장 이력(참여공종 포함) 목록
근로계약서 조회	특정 조건으로 근로계약서를 검색하고 내용을 확인하는 기능. <br> * 검색 필드: 현장, 날짜, 노무자 이름, 참여공종, 참여구간(작업위치) <br> * 상세 조회: 날짜, 현장명, 참여공종, 참여구간 표기 및 계약서 원문 조회
현장별 출역현황 조회	전체 또는 특정 현장의 일자별 출역 현황을 조회하는 기능. <br> * 조회 필드: 현장, 날짜, 노무자, 출근시간, 퇴근시간, 공수, 산정비용
현장별 공사일보 조회	각 현장에서 작성된 공사일보를 조회하여 공정 현황을 확인하는 기능. <br> * 목록 조회 필드: 현장, 날짜, 출역인원, 공정율 <br> * 상세 조회: 날씨, 금일작업내용, 명일작업계획, 공종별 출역인원(공종, 전일누계, 금일출역, 출역누계), 장비현황(장비구분, 전일누계, 금일투입, 전체누계)
공종/장비/노무단가 관리	회사 차원에서 사용되는 표준 공종, 장비, 공종별 노무 기준 단가를 생성하고 수정하는 기능.
대시보드	전사적 관점의 핵심 지표를 시각화하여 제공. <br> * 주요 지표: 현장별 공정율 현황, 현장별 출역현황(예정/실제), 현장별 근로계약 현황(당일 출역/서명)

3.2. 현장 관리자 (Site Manager)

현장 관리자는 개별 현장의 운영 효율성을 극대화하고 실시간 변수에 신속하게 대응하는 실무 책임자입니다. 기능들은 현장 중심의 정보 관리, 인력 배정, 출역 통제 및 일일 업무 보고에 초점을 맞춥니다.

기능	주요 명세
현장정보 관리	담당 현장의 상세 정보를 관리하는 기능. <br> * 관리 필드: 현장명, 안면인식기 고유번호, 현장주소, 현장 연락처, 참여공종, 발주처, 도급업체, 출근/퇴근 기준시간, 휴게/식사시간
노무팀장 등록 (초대)	현장에 투입될 노무팀장을 시스템에 등록하거나 초대하는 기능. <br> * 프로세스: 기등록된 노무팀장 목록에서 선택하거나, 신규 등록 시 이름, 주민번호, 핸드폰번호, 이메일 정보 입력 후 초대 (초대 시 SMS/알림톡/이메일 전송)
팀 관리	노무팀을 특정 작업에 배정하고 상태를 관리하는 기능. <br> * 조회 필드: 팀명, 팀장명, 연락처, 담당공종, 팀원수, 작업기간, 작업구간, 수락여부 <br> * 핵심 프로세스: <br> 1. 팀장 수락 전: 담당공종, 작업기간, 작업구간 수정 가능 <br> 2. 팀장이 거부 시: '수락거부'로 표기되고 '재요청' 버튼 활성화 <br> 3. 팀장이 수락 시: '저장' 버튼 활성화
출역현황 관리 (조회 및 수정)	현장 인원의 출역 기록을 관리하고 예외 사항을 처리하는 기능. <br> * 조회 필드: 날짜, 공종, 노무자, 연락처, 출근시간, 퇴근시간, 공수, 단가, 일노임, 출역변경사항 <br> * 필터링 기능: 날짜, 팀, 공종으로 데이터 필터링 가능 <br> * 수정 가능 필드: 출근시간, 퇴근시간, 공수, 단가 <br> * 핵심 프로세스: 데이터 수정 시, '출역변경사항' 필드에 사유 기재 필수. <br> * 편의 기능: 선택박스를 이용한 공수, 단가 등 일괄 변경 기능 제공.
작업일지 조회 및 출역 통제	노무팀장이 제출한 작업일지를 기반으로 익일 작업을 관리하는 기능. <br> * 목록 조회 필드: 날짜, 공종, 작업위치, 작업내용, 노무팀장, 연락처 <br> * 상세 조회: 금일작업내용, 명일작업계획, 금일/명일 출역인원 <br> * 출역 통제: 명일 출역 계획 인원 중 특정 인원을 선택하여 '출역 금지' 처리 가능. (사유 입력 필수, 저장 시 노무팀장에게 알림 전송)
작업일보 관리	현장의 일일 공사일보를 작성, 조회, 승인하는 기능. <br> * 데이터 연동: <br>   - 날씨/기온: 기상정보와 연동하여 자동 입력 (수정 가능). <br>   - 금일/명일 작업 내용: 노무팀장 작업일지를 '취합' 버튼으로 불러옴 (기존 내용 초기화 시 알림 필수). <br>   - 출역 인원: 출역 현황에서 공종별 인원 자동 취합. <br> * 입력 필드: 장비출역 현황. <br> * 규칙: 최종 승인 후 수정 불가.
근로자 정보 조회	현장에 투입된 근로자의 정보를 조회하는 기능. <br> * 목록 조회 필드: 이름, 핸드폰번호, 이메일, 마지막 참여공종 <br> * 상세 조회: 기본 상세정보 및 해당 현장 투입 날짜, 참여공종 목록
근로계약서 관리	근로계약서 현황을 조회하고 서명을 요청하는 기능. <br> * 조회 필드: 날짜, 노무자 이름, 참여공종, 참여구간, 서명수/계약서수 <br> * 일괄 서명요청: 미서명자에게 일괄적으로 서명 요청(SMS/알림톡/이메일) 전송 기능.
작업구간 관리	현장 내 작업 구간(위치) 및 관련 작업 내용을 정의하는 기능. <br> * 관리 필드: 작업구간명, 공종, 작업위치, 작업내용
현장별 공종 및 장비 관리	본사 기준 정보 외에 현장에서만 사용하는 공종이나 장비를 추가 관리하는 기능.
대시보드	담당 현장의 핵심 운영 지표를 시각적으로 제공. <br> * 주요 지표: 출역현황, 근로계약서 미서명 수(일괄 서명요청 버튼 포함), 공종별 출역현황, 팀별 작업 배정 현황

3.3. 노무팀장 (Labor Team Lead)

노무팀장은 소속 팀원을 효율적으로 관리하고 현장과의 소통을 원활하게 하는 중간 관리자입니다. 기능들은 팀원 관리, 현장 작업 수락, 팀의 출역 현황 보고 및 권익 보호에 중점을 둡니다.

기능	주요 명세
팀원 관리 (등록, 초대, 제거)	소속 팀원을 관리하는 기능. <br> * 초대: 이름, 주민번호, 핸드폰번호, 이메일 정보로 신규 팀원 초대 (초대 시 SMS/알림톡/이메일 전송). <br> * 중복 처리: 동일한 이름/주민번호 입력 시 기존 사용자 정보 확인 후 추가 여부 결정. <br> * 상태 관리: 팀원의 수락/거부 상태 확인. 거부 시 '재요청' 버튼 활성화. <br> * 제거: 선택한 팀원을 팀에서 제거.
현장별 팀 관리 (작업 수락/거부)	현장 관리자로부터 배정받은 작업을 관리하는 기능. <br> * 조회 필드: 현장명, 참여 팀명, 참여기간, 참여공종, 요청 노무자 수, 포함 팀원, 수락/거부 버튼 <br> * 프로세스: <br>   1. 작업 요청 확인 후, '포함되는 팀원' 영역에서 내 팀원 선택. <br>   2. '수락' 또는 '거부' 버튼으로 작업 참여 여부 결정. <br>   3. 거부 시 목록에서 제거됨.
출역현황 관리 (조회 및 이의제기)	소속 팀원의 출역 현황을 확인하고 오류를 시정 요청하는 기능. <br> * 조회 필드: 날짜, 현장명, 공종명, 이름, 연락처, 출근/퇴근시간, 공수, 비고, 근로계약서 서명여부 <br> * 핵심 비즈니스 규칙: 팀원의 공수가 1 미만일 경우(결근, 지각, 조퇴 등) '이의제기' 버튼이 활성화되며, 클릭 시 이의 내용을 작성하여 현장 관리자에게 알림 전송. <br> * 편의 기능: 미서명자를 선택하여 근로계약서 '일괄 서명요청' 가능.
대시보드	팀 관리 핵심 지표를 요약하여 제공. <br> * 주요 지표: 실시간 팀원 출역현황(출역/예정/출역률), 안면인식 미등록 현황, 근로계약서 미서명 현황, 간편 작업일보 작성

3.4. 노무자 (Worker/Laborer)

노무자는 자신의 근로 기록에 대한 투명한 정보를 제공받고, 계약 관련 사항을 직접 확인하는 최종 사용자입니다. 기능들은 정보 접근성, 편의성, 그리고 신뢰 관계 구축에 중점을 둡니다.

기능	주요 명세
출역현황 조회	자신의 기간별 출역 기록을 상세 내역과 함께 조회하는 기능. <br> * 조회 필드: 날짜, 현장, 팀, 공종, 공수, 단가, 일노임, 팀장명, 팀장연락처, 비고, 근로계약서, 이의제기 버튼 <br> * 요약 정보: 조회된 기간의 총 출역일, 총공수, 평균단가, 총노임을 화면 상단에 요약하여 표시. <br> * 조회 기간: 기본 1개월, 기간 설정하여 조회 가능.
대시보드 (월간 현황 및 근로계약)	월간 근로 현황과 계약 상태를 한눈에 파악하는 기능. <br> * 월간 출역 현황: 캘린더 형식으로 일별 공수 표기. <br> * 근로계약 관리: 서명이 필요한 근로계약서 목록 확인 및 전자 서명 진행.

각 사용자에게 최적화된 기능들이 유기적으로 작동하기 위해서는 모든 사용자가 공통으로 사용하는 기반 기능들이 필요합니다. 이는 서비스의 일관성과 사용 편의성을 보장하는 중요한 요소입니다.

4. 공통 기능 요구사항 (Common Feature Requirements)

모든 사용자에게 일관된 경험을 제공하고 서비스의 기본 편의성을 보장하기 위해 역할과 무관하게 제공되는 공통 기능이 필요합니다. 이 기능들은 사용자 역할 간의 정보 흐름을 촉진하고, 시스템 전반의 사용성을 높이며, 사용자가 자신의 정보를 주도적으로 관리할 수 있도록 지원합니다.

* 알림 (Notifications)
  * 목적: 사용자가 자신의 역할과 관련된 주요 이벤트나 요청을 놓치지 않도록 실시간으로 정보를 전달합니다.
  * 핵심 요구사항:
    * 사용자 초대, 작업 배정 요청, 작업 수락/거부, 출역 금지 통보, 이의제기 접수 등 주요 프로세스 발생 시 관련자에게 즉시 알림(SMS/알림톡/이메일 또는 인앱 알림)을 전송해야 합니다.
    * 사용자는 '알림함' 메뉴를 통해 수신한 모든 알림 내역을 목록으로 확인할 수 있어야 합니다.
* 내 정보 관리 (Profile Management)
  * 목적: 사용자가 자신의 개인정보 및 근로계약에 필요한 정보를 직접 조회하고 최신 상태로 유지할 수 있도록 합니다.
  * 핵심 요구사항:
    * 모든 사용자는 이름, 연락처, 이메일 등 개인정보를 조회하고 수정할 수 있어야 합니다.
    * 노무자는 근로계약 체결에 필요한 상세 정보(주민번호 등)를 관리할 수 있어야 합니다.
* 공지사항 (Announcements)
  * 목적: 관리자가 특정 그룹 또는 전체 사용자를 대상으로 중요 정보를 효과적으로 전파하는 채널을 제공합니다.
  * 핵심 요구사항:
    * 본사 관리자는 전체 사용자를 대상으로, 현장 관리자는 자신이 속한 현장의 사용자를 대상으로 공지사항을 게시할 수 있어야 합니다.
    * 모든 사용자는 자신이 소속된 회사 또는 현장의 공지사항을 조회할 수 있어야 합니다.
* 안면인식 연동 (Facial Recognition Integration)
  * 목적: 안면인식 출퇴근 기록 시스템과 연동하여 출역 데이터의 정확성과 신뢰성을 확보합니다.
  * 핵심 요구사항:
    * 프로세스:
      1. 사용자 정보 등록: 현장관리자, 노무팀장, 노무자가 현장에 초대되거나 팀에 등록될 때 사진을 포함한 사용자 정보가 안면인식 서버로 전송되어 등록됩니다.
      2. 활성 사용자 전송: 참여 기간이 시작되는 노무자 정보는 매일 자정 배치(batch) 작업을 통해 안면인식 서버로 자동 전송됩니다.
      3. 사용자 정보 삭제: 현장에서의 참여 기간이 종료되면, 해당 사용자 정보는 안면인식 서버에서 삭제 처리됩니다.
* 로그인 및 접근 제어 (Login & Access Control)
  * 목적: 사용자의 역할과 소속에 따라 안전하고 편리한 인증 방식을 제공하고, 인가된 정보에만 접근할 수 있도록 권한을 제어합니다.
  * 핵심 요구사항:
    * 인증 방식 분리: 로그인은 '본사'와 '현장' 탭으로 구분되어야 합니다. 본사 관리자는 사업자등록번호와 비밀번호로, 현장 관련 사용자(현장관리자, 노무팀장, 노무자)는 소셜/간편인증으로 로그인해야 합니다.
    * 사용자 로그인 시나리오:
      1. 사용자가 소셜/간편인증으로 로그인합니다.
      2. 시스템은 사용자가 소속된 현장이 여러 개인지 확인합니다.
      3. 소속 현장이 1개인 경우, 해당 현장의 권한 선택 화면으로 즉시 이동합니다.
      4. 소속 현장이 여러 개인 경우, 먼저 현장 선택 화면을 제시한 후, 선택된 현장의 권한 선택 화면으로 이동합니다.
    * 다중 권한 처리: 한 현장 내에서 사용자가 2개 이상의 권한(예: 현장관리자 겸 노무팀장)을 가질 경우, 어떤 권한으로 접속할지 선택하는 화면을 제공해야 합니다.
    * 모든 접속은 단일 도메인을 통해 이루어져야 합니다.

이러한 기능적 요구사항을 안정적으로 지원하기 위해서는 서비스의 품질과 확장성을 결정하는 기술적 기반, 즉 비기능적 요구사항이 명확하게 정의되어야 합니다.

5. 시스템 아키텍처 및 비기능 요구사항 (System Architecture & Non-Functional Requirements)

성공적인 SaaS 서비스는 잘 설계된 기능뿐만 아니라, 안정성, 확장성, 사용성을 보장하는 견고한 기술적 토대 위에 구축됩니다. 본 섹션에서는 서비스의 품질 속성을 정의하는 비기능적 요구사항을 명시합니다. 이는 장기적인 서비스 운영과 유지보수의 효율성을 결정하는 중요한 기준이 됩니다.

* SaaS 아키텍처 (멀티테넌시):
  * 시스템은 여러 고객사(건설사)가 단일 소프트웨어 인스턴스를 공유하는 멀티테넌트(Multi-tenant) 아키텍처를 지원해야 합니다.
  * 각 테넌트(고객사)의 데이터는 논리적으로 완벽하게 격리되어 데이터 보안 및 프라이버시를 보장해야 합니다.
* 모듈식 아키텍처 (Modular Architecture):
  * 서비스의 확장성을 보장하기 위해, 시스템은 모든 고객에게 공통적으로 제공되는 '코어 기능'과 선택적으로 추가될 수 있는 '확장 기능'을 구분하여 모듈식으로 설계해야 합니다.
* 반응형 웹 디자인 (Responsive Web Design):
  * 서비스의 모든 웹 페이지는 PC, 태블릿, 모바일 등 다양한 디바이스의 화면 크기와 해상도에 자동으로 최적화되는 반응형으로 구현되어야 합니다.
  * 이를 통해 사용자는 어떤 환경에서든 일관되고 쾌적한 사용자 경험을 누릴 수 있어야 합니다.
* 기술 스택 (Technology Stack):
  * 데이터베이스 관리 시스템(DBMS)은 라이선스 비용 부담이 적고 커뮤니티가 활성화된 오픈소스 기반으로 선정해야 합니다.
  * 전체 기술 스택은 최신 기술 트렌드를 반영하여 향후 기능 확장 및 유지보수가 용이하도록 설계되어야 합니다.
* 사용자 경험 (User Experience, UX):
  * 모든 UI/UX는 사용자가 별도의 교육이나 매뉴얼 없이도 서비스를 직관적으로 이해하고 사용할 수 있도록 설계되어야 합니다.
  * 복잡한 건설 현장 업무를 최대한 단순하고 명확하게 시스템에 반영하여 사용자의 학습 곡선을 최소화하는 것을 목표로 합니다.
* 모바일 앱 연동 (Deep Linking):
  * 신규 사용자가 SMS/알림톡 등을 통해 최초 초대를 받을 경우, 앱이 설치되지 않았다면 스토어로 안내하고, 설치 후에는 로그인 또는 특정 콘텐츠 화면으로 바로 연결되는 딥링크(Deep Link) 기능이 구현되어야 합니다.
* 다국어지원: 한국어와 영어를 기본으로 지원해야 하고, 향후 기타언어 추가를 위한 기반도 마련이 되어야 한다.

본 기획서는 성공적인 건설현장 인력관리 SaaS 서비스 개발을 위한 청사진입니다. 여기에 명시된 요구사항들은 개발팀, 기획팀, 그리고 모든 프로젝트 이해관계자 간의 명확한 소통의 기준이 되어, 사용자의 기대를 뛰어넘는 가치 있는 서비스를 만들어나가는 길잡이가 될 것입니다.
