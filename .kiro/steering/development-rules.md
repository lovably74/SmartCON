# 개발 규칙 (Development Rules)

## 언어 및 문서화 규칙
- **모든 응답은 한국어로 작성**
- **한국어로 주석 작성, 내용이 바뀌면 주석도 변경**
- **디버그 로그는 문제가 해결되었다면 반드시 삭제**

## 문서 관리 규칙
- **문서는 항상 /docs 폴더 하위에 저장. 없으면 생성**
- **문서 변경시 /docs/backup 폴더에 히스토리 문서 백업**
- **백업 형식**: `문서명_버전_년도월일시분초.md` 
- **예시**: `문서명.md` → `문서명_V01_20251217183401.md`

## 파일 인코딩 규칙
- **파워쉘용 스크립트는 ANSI 코드로 작성하며, 한글은 가급적 사용하지 말것**
- **파워쉘용 스크립트를 제외한 모든 파일은 UTF-8 without BOM 형태로 저장**

## 보안 및 코드 품질 규칙
- **보안에 취약한 코드는 특별한 지시가 없다면 사용 금지**
- **절대로 추측성 수정은 하지마세요. 원인을 모르면 수정 금지**
- **간단하지 않은 문제라면 웹 검색을 통해 최신 해결책을 검색**

## 프로덕션 환경 규칙
- **이 프로젝트는 프로덕션입니다. 오류 수정을 위해 우회하는 임시 코드를 작성 금지**
- **기능 추가나 변경이 되면 테스트 코드 작성 및 업데이트**
- **작업이 끝나면 테스트 실행**

## 문제 해결 접근법
1. 원인 파악을 위한 충분한 조사
2. 웹 검색을 통한 최신 해결책 확인
3. 근본적인 해결책 적용 (임시방편 금지)
4. 테스트 코드 작성/업데이트
5. 테스트 실행으로 검증

## 코드 작성 원칙
- 명확하고 이해하기 쉬운 한국어 주석
- 보안 모범 사례 준수
- 테스트 가능한 코드 구조
- 프로덕션 품질의 안정적인 구현
